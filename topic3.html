Git Essentials

Version Control System for Tracking Code

What is Git?

Git is a distributed version control system used to track changes in source code during software development. It helps teams collaborate efficiently.

Key Concepts

  • Repository (Repo): Storage for your project and its history
  • Commit: Snapshot of changes
  • Branch: Separate line of development
  • Merge: Combine branches
  • Remote: A hosted repository (e.g., GitHub)

Git Workflow

The common Git workflow:

  1. Clone repository
  2. Create a branch
  3. Make changes and commit
  4. Push branch to remote
  5. Create pull request and merge

Common Git Commands

  • git init - Initialize a new repository
  • git clone [url] - Copy a repository
  • git status - Check changes
  • git add [file] - Stage changes
  • git commit -m "message" - Save snapshot
  • git push - Send changes to remote
  • git pull - Fetch changes from remote

Branching & Merging

Branching allows separate development. Merging integrates changes back:

  • git branch [name] - Create a branch
  • git checkout [name] - Switch branch
  • git merge [branch] - Merge a branch into current
  • git log - View commit history

Remote Repositories

Remote repositories like GitHub, GitLab, or Bitbucket store your project online. You can push local changes, collaborate, and track history.

  • git remote add origin [url] - Link local repo to remote
  • git push -u origin main - Push initial commit
  • git pull - Fetch and merge remote changes

Git Best Practices

  • Commit often with meaningful messages
  • Use branches for new features or fixes
  • Pull latest changes before starting work
  • Review pull requests carefully
  • Keep repository clean and organized
